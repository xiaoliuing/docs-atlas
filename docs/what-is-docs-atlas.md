# 什么是 Docs Atlas

`Docs Atlas` 是一个本地 Markdown 文档聚合阅读器。

它的目标不是在线协作，也不是后台内容管理，而是把你分散在不同项目目录中的 Markdown 文档聚合到一个统一入口里查看。

当前项目同时提供：

- Web 端
- 桌面端

两者共享同一套文档组织规则和阅读能力。

## 它适合什么场景

- 你有多个项目，每个项目都有自己的设计文档
- 你会把教程、说明、接入文档写成 Markdown
- 你希望用一个统一界面查看这些文档
- 你希望 AI 生成的文档也能直接落地使用

## 它解决什么问题

常见问题通常是：

- 文档散落在不同仓库
- 阅读入口不统一
- 搜索范围只在单个项目内
- 设计文档和教程文档切换成本高

`Docs Atlas` 通过“多目录聚合 + 左侧导航 + 全文搜索 + 统一阅读界面”来解决这些问题。

## 它的核心规则

- 默认读取 `docs/`
- 也可以通过 `config.yaml` 聚合多个文档目录
- 一级目录视为专题
- 专题里的 `README.md` 是入口页
- 根目录下直接放置的 Markdown 会直接显示为文档，不额外包一层目录

## 这篇文档为什么放在根目录

这篇文件故意放在 `docs/` 根目录下，不放进子目录。

这样可以演示一个规则：

- 如果某个文档源根目录下直接就是 Markdown 文件
- 它会在左侧菜单里直接显示为一个文档项
- 不会再额外包一层空目录

这对“项目说明”、“聚合首页补充说明”这类文档很有用。
