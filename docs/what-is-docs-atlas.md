# 什么是 Docs Atlas

`Docs Atlas` 是一个以桌面端为主的本地 Markdown 文档聚合系统。

它面向的不是在线协作场景，而是开发者本机长期积累的设计文档、教程文档、接入说明和 AI 生成文档。项目的核心目标很直接：

- 把分散在不同目录、不同项目里的文档聚合到一个入口
- 保持目录结构清晰，减少“文档在哪”的查找成本
- 在阅读和搜索的基础上，直接在桌面端修改 Markdown 正文

## 当前产品形态

目前项目包含两个端：

- 桌面端：主产品，负责聚合、阅读、搜索、编辑和本地管理
- Web 端：补充能力，负责只读预览、静态构建和部署分享

如果你第一次接触这个项目，建议优先从桌面端开始。

## 它解决什么问题

现实中的文档管理常见几个问题：

- 每个项目都有自己的 `docs/`，入口分散
- 架构设计、模块说明、运维文档分布在不同仓库
- AI 能快速生成文档，但后续阅读、修订和归档很麻烦
- 过一段时间再找一篇文档，经常想不起来它属于哪个项目

Docs Atlas 的做法是把这些问题统一到一套阅读和维护流程里。

## 当前已经支持的核心能力

### 桌面端

- 多个文档仓库
- 每个文档仓库挂载多个文档源
- 文档源支持分组和嵌套
- 左侧目录树导航
- 全局搜索和当前文档仓库搜索
- 阅读位置记忆和最近阅读恢复
- 收藏、最近阅读
- 主题模式和多主题色切换
- 文档图片预览、缩放、切换
- Markdown 正文直接编辑
- 自动保存和快捷键保存
- 标题区显示最后编辑时间
- Mermaid 流程图渲染

### Web 端

- 基于本地 Markdown 目录生成文档站
- 支持 `config.yaml` 聚合多个文档源
- 支持静态构建和 GitHub Pages 部署
- 支持图片资源复制和相对路径解析

## 文档规则

无论桌面端还是 Web 端，建议尽量遵循同一套内容结构：

- 一级专题目录使用 `README.md` 作为入口页
- 其他 `.md` 文件作为正文文档
- 根目录下直接放置的 `.md` 文件会直接作为文档项显示
- 文档标题使用首个 `# 一级标题`
- 图片使用相对路径引用
- 正文尽量以 `##`、`###` 组织层级

## 这个项目现在更像什么

你可以把 Docs Atlas 理解成一个偏开发者工作流的本地知识库入口，而不是传统 CMS：

- 它不做后台发布流程
- 它不依赖远端数据库
- 它优先复用你已经存在的 Markdown 目录
- 它强调聚合、阅读、搜索、修订

对 AI Coding 时代来说，这个定位很重要：文档越来越容易生成，但更需要一个稳定的本地入口来沉淀和回看。
