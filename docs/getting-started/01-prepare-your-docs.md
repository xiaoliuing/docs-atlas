# 下载桌面端并完成首次启动

如果你第一次使用 `Docs Atlas`，建议直接从桌面端开始。

## 下载地址

- Latest Release: https://github.com/xiaoliuing/docs-atlas/releases/latest
- Releases 页面: https://github.com/xiaoliuing/docs-atlas/releases

当前发布包通常包含这些平台：

- macOS Apple Silicon
- Windows x64
- Linux x64

具体可下载文件以后续 release 中的产物为准。

## 首次启动后会看到什么

桌面端第一次打开时，会带一个默认文档仓库，指向项目内置的示例 `docs/` 目录。

这个默认示例不是临时页面，而是为了让你直接体验完整流程：

- 左侧目录树
- 文档阅读
- 搜索
- 图片预览
- Mermaid 渲染
- Markdown 正文编辑
- 自动保存和手动保存

也就是说，应用第一次启动后就已经是可用状态，不需要先配一堆外部目录。

## 什么是文档仓库

文档仓库是桌面端的核心概念。

一个文档仓库可以理解为：

- 一组文档源的集合
- 一套独立的阅读上下文
- 一个长期保存的本地知识入口

每个文档仓库可以拥有：

- 自己的名称
- 自己的主题色
- 自己的文档源树
- 自己上次阅读的文档和滚动位置

## 桌面端编辑器能力

当前桌面端的文档区域已经不是纯预览，而是可编辑的 Markdown 编辑器。

你可以直接在文档正文中：

- 修改标题和段落
- 编辑代码块、表格、引用块
- 查看 Mermaid 图表渲染效果
- 使用 `Ctrl + S` 或 `Cmd + S` 手动保存

同时还支持：

- 自动保存
- 切换文档前自动落盘
- 标题区域显示最后编辑时间

这意味着 Docs Atlas 现在不仅是阅读器，也可以承担轻量的本地文档维护工作。

## 首次启动建议先做什么

推荐按下面的顺序体验：

1. 先打开示例文档看看目录结构
2. 用搜索试一下跨文档查找
3. 打开一篇文档直接修改几行内容
4. 用快捷键保存，确认能写回本地文件
5. 再开始创建你自己的文档仓库

## macOS 需要注意的点

如果当前下载包没有做 Apple 签名和 notarization，macOS 首次打开时可能会出现系统安全提示，需要手动放行。

这属于平台安全限制，不影响应用本身的文档读取和编辑能力。
