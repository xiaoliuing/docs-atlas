# 用 AI 生成可直接接入的文档

`Docs Atlas` 不只是一个阅读界面，也很适合作为 AI 文档生成后的承载层。

## 推荐工作流

1. 先确定主题，例如 `FastAPI`、`Redis`、`Kubernetes`
2. 用提示词让 AI 输出一整组 Markdown 文档
3. 把文档放进某个本地 docs 目录
4. 用 `Docs Atlas` 聚合展示

## 生成规则

给 AI 的要求尽量简单明确：

- 输出一个主题目录
- 目录下必须有 `README.md`
- 再输出若干篇正文文档
- 每篇文档以 `# 一级标题` 开头
- 正文主要使用 `##` 和 `###`
- 代码块带语言标识

## 示例提示

```text
请为主题“FastAPI 项目设计文档”生成一组适合 Docs Atlas 的 Markdown 文档。

要求：
1. 输出 docs/fastapi/ 目录
2. 必须包含 README.md
3. 再输出 5 篇正文文档
4. 每篇文档必须有完整正文，不要只给提纲
5. 内容面向开发者，偏设计说明和接入文档
```

## 为什么这种方式有价值

在 AI coding 场景里，代码和文档都在快速生成。

如果没有统一的文档入口，最终会出现：

- 各项目说明分散
- 设计文档难以统一查看
- 生成结果难以复用

`Docs Atlas` 的价值就在这里：

- AI 负责生成
- 你负责筛选和整理
- `Docs Atlas` 负责聚合和展示
