02. Build Your First Static Module

这一页的目标是先把用户注意力重新拉回正文。左侧目录只服务于跳转，不再把最近阅读、收藏、工作空间说明堆成一大坨。

顶部搜索和设置继续放在 title bar。最近阅读和收藏虽然是全局页，但只保留为一级切换，不抢目录高度。

const layout = { globalTabs: ['阅读', '最近阅读', '收藏'], workspace: 'compact-context', directory: 'primary', }

如果切到“最近阅读”或“收藏”，左侧目录区域可以切成全局说明卡或相关筛选，不需要继续展示目录树。

优点

  • 改动成本最小，可以直接在现有桌面端上收敛。
  • 目录区回到主位，用户打开应用后第一眼就是文档树。
  • 最近阅读 / 收藏仍然全局化，但入口不抢空间。

风险

  • 左栏里依然承担 3 个层级，只是已经明显压缩。
  • 未来如果再加标签、任务、AI，左栏仍可能再度膨胀。
