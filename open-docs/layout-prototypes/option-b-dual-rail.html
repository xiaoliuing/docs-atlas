01. 项目架构

这里的核心是把结构关系讲清楚：功能轨道是全局，目录是工作空间，正文是文档本身。三者不再混在一列里。

当你切换到最近阅读或收藏时，只需要把中间的“目录栏”切成对应列表或者筛选器，最左边的全局轨道不动。

global rail -> 阅读 / 最近 / 收藏 directory pane -> 当前工作空间的目录树 content pane -> 文档 / 列表 / 设置详情

这会让桌面端的扩展性明显更好，未来加入标签、任务、同步、智能问答时，也不容易把目录区再挤乱。

优点

  • 信息架构最清晰，目录被完整保护下来。
  • 后续可扩展性最好，适合桌面端长期演进。
  • 最近阅读 / 收藏天然就是全局层，不会再和工作空间打架。

风险

  • 需要改动桌面端框架布局，工作量比方案 A 大。
  • 最左边轨道是新交互，需要控制好图标和提示文案。
