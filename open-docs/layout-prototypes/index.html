Docs Atlas Desktop 页面排版提案

这 3 套方案只讨论信息层级、页面切换和模块占位，不直接改现有桌面端代码。核心目标是把“文档目录”重新拉回主角位置，同时保留当前文档仓库、最近阅读、收藏、搜索、设置这些能力。

推荐优先看

方案 A · 目录优先

左侧还是一个栏，但把全局入口压成一排，文档仓库压成一张很薄的卡片，剩余空间全部让给目录树。

  • 适合当前桌面端，改造成本最低。
  • 用户一打开就先看到目录，不会再被文档仓库信息往下顶。
  • 最近阅读和收藏保留为全局页，但入口不喧宾夺主。

层级最清晰

方案 B · 双层导航

用极窄功能轨道承接全局入口，目录单独占一整栏。全局导航、文档仓库、目录三种层级被完全拆开。

  • 目录区非常纯粹，阅读型产品感更强。
  • 适合未来功能继续增加。
  • 改动稍大，需要重构桌面端框架布局。

最克制

方案 C · 阅读优先

文档仓库更多上移到顶部上下文区，左侧几乎只保留目录。适合把应用继续往“阅读器 / 知识库”方向推进。

  • 界面最干净，视觉压力最低。
  • 目录权重最高。
  • 文档仓库操作会更依赖顶部入口。

我当前的建议

如果你要的是“不推翻现有能力、但马上把页面理顺”，优先选 方案 A。它最符合你刚才提出的问题：文档仓库只保留当前文档仓库能力，但不再压住目录。

如果你更希望桌面端长期往专业知识库工具走，方案 B 的信息架构更稳，后续再加标签、任务、智能问答、文档源管理也更好扩展。

本轮只看什么

  • 最左侧放什么，是否抢走目录空间。
  • 最近阅读 / 收藏是否应该全局化。
  • 文档仓库切换和文档仓库设置应该出现在什么位置。
  • 顶部搜索、设置、正文、大纲之间是否顺手。
