02. Router 与 Docs

这套方案的优点是极其干净。用户最关心的目录树不再和任何文档仓库说明争夺左栏高度。

文档仓库能力并没有消失，而是被上移到了正文区域顶部的上下文带里。这样在阅读时能看到当前文档仓库，但目录仍保持完整。

header -> 全局页 + 搜索 + 系统设置 left -> 目录 body -> 当前文档仓库上下文 + 正文 right -> 大纲

如果要把桌面端继续做成“本地知识库浏览器”，这是视觉上最像成熟产品的一套。

优点

  • 视觉最轻，目录权重最高。
  • 桌面端更像一个真正的阅读器 / 知识库应用。
  • 正文区更自然承接“当前文档仓库上下文”。

风险

  • 文档仓库操作上移后，需要重新定义顶部与正文头部的职责。
  • 比方案 A 更抽象，第一次上手需要更强的交互引导。
