# Docs Atlas Desktop Phase 2 Remaining Plan

## 1. 目标

这份文档用于收拢第二阶段剩余工作，并明确接下来的实现顺序。

当前桌面端主骨架已经可用，但还缺少“产品化收口”。本阶段后续工作不再做大范围架构调整，而是围绕这些方向补齐：

1. 设置中心收口
2. 搜索能力补齐
3. 阅读辅助能力补齐
4. 错误处理与恢复
5. 数据管理与日志能力

`LLM / RAG` 相关能力明确放到下一阶段，不纳入本阶段。

## 2. 当前判断

桌面端现在不是不可用，而是：

- 核心阅读链路已打通
- 工作空间与文档源管理已打通
- 发布链路已打通
- 但应用信息架构仍偏“单页壳 + 浮层补丁”

这会直接限制后续继续添加：

- 系统设置
- 工作空间设置
- 数据与日志入口
- 最近阅读 / 收藏 / 视图入口

所以第二阶段收尾的第一项，不应该先继续堆功能，而应该先把桌面端从“单页壳”升级为“应用内多视图”。

## 3. 信息架构调整

## 3.1 结论

需要做多视图，但当前不必先上完整路由系统。

第一步建议采用应用内 `view state`：

- `reader`
- `settings`

后续如果再引入：

- `favorites`
- `recent`
- `workspace-overview`

再评估是否切到显式路由。

## 3.2 原因

当前设置入口是标题栏浮层，只适合少量配置，不适合继续承接：

- 系统设置
- 工作空间设置
- 数据目录 / 日志
- 后续错误恢复和诊断入口

而这些内容本质上都更适合作为完整页面，而不是临时浮层。

## 4. 第一批页面设计

## 4.1 Reader View

职责：

- 保留当前三栏阅读结构
- 继续承载目录、正文、大纲、搜索结果跳转
- 不承担设置与管理逻辑

组成：

- `DesktopDocsSidebar`
- `DesktopDocReader`
- `DesktopDocToc`

## 4.2 Settings View

职责：

- 承载系统设置、工作空间设置、数据与日志入口
- 成为第二阶段剩余产品化能力的收口点

建议布局：

- 左侧：设置导航
- 右侧：当前设置分组内容

第一批分组：

1. `外观`
2. `工作空间`
3. `数据与日志`

## 4.3 Settings View 内部结构

### 外观

放置：

- 跟随系统 / 浅色 / 深色
- 主题色切换
- 当前主题说明

### 工作空间

放置：

- 当前工作空间摘要
- 修改工作空间元信息入口
- 管理文档源入口
- 导入 / 导出工作空间
- 新建工作空间入口

### 数据与日志

放置：

- 打开应用数据目录
- 打开日志目录
- 导出日志文件
- 后续索引状态、错误恢复、健康检查入口预留

## 5. 第一批组件边界

### `DesktopAppShell`

职责：

- 桌面应用顶层编排
- 管理当前主视图 `reader/settings`
- 继续承载标题栏搜索与全局动作

### `DesktopSettingsView`

职责：

- 设置中心页面容器
- 组织设置分组导航和内容面板

Props:

- 当前工作空间摘要
- 主题设置状态
- 数据目录 / 日志动作状态

Emits:

- 打开工作空间编辑
- 打开文档源编辑
- 新建工作空间
- 导入 / 导出工作空间
- 修改主题模式
- 修改主题色
- 打开数据目录 / 日志目录 / 导出日志

### `DesktopSettingsNav`

职责：

- 渲染设置分组导航
- 切换当前设置分组

### `DesktopAppearanceSettingsPanel`

职责：

- 渲染主题模式和主题色设置

### `DesktopWorkspaceSettingsPanel`

职责：

- 渲染当前工作空间信息与操作入口

### `DesktopDataSettingsPanel`

职责：

- 渲染数据目录、日志目录、导出日志和后续诊断入口

## 6. 实施顺序

### Milestone 1: 设置中心收口

1. 增加 `reader/settings` 主视图状态
2. 把设置从浮层改成完整视图
3. 新增设置中心及其导航
4. 拆分外观、工作空间、数据日志三块面板

### Milestone 2: 搜索与阅读增强

1. 搜索筛选维度补齐
2. 最近阅读视图
3. 收藏 / 书签

### Milestone 3: 错误处理与恢复

1. 文档源异常提示
2. 索引失败提示
3. 恢复入口

### Milestone 4: 数据管理与产品化细节

1. 数据目录管理完善
2. 日志导出完善
3. 健康检查与诊断入口预留

## 7. 本轮执行范围

当前这一轮代码实现先覆盖：

1. 新增 `Settings View`
2. 完成第一批设置中心组件拆分
3. 把现有系统设置浮层迁移为页面
4. 整合工作空间操作入口
5. 保持现有阅读视图不退化
